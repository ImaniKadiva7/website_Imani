REAL TIME EMBEDDED SYSTEM PROTOTYPE

Introducing my Real-Time Embedded System Prototype. Unlock unparalleled efficiency and precision as tasks harmoniously unfold concurrently, ensuring optimal resource utilization and responsiveness. This project showcases the power of real-time multitasking, useful in industries like automotive, manufucturing, health care and beyond.

See the video of the working system below.

This project demonstrates my skills and experience in writing code that can do the following:

  1. Control and run multiple processes concurrently on a single microcontroller in a non-blocking fashion.
  2. Controlling digital pins of ATMEGA328p microcontroller in a Bare metal fashion.
  3. I2C interfacing between two modules. i.e Microcontroller and MPU_6050 integrated circuit
  4. Driving and interrogating I2C device.
  5. How to deal with situations where there are insufficient port-pins.
  6. Driving I/O lines in sequence according to their timing diagrams.
  7. Dealing with noise from the Pushbutton by registering correct button press after 300ms.

HOW THE SYSTEM WORKS

The system has got 3 modes all running concurrently in a non-blocking fashion. The first mode in the list below works initially when the system is powered. The push button on the left side of the video is used to change to other modes of the system and the push button on the right side in the video is used to display numbers and letters on the 7 segment display.


Consider the following modes of the system.

  1. Traffic lights Contoller module is on and the 7 segment displays letter "t".
  2. Traffic lights controller module is on and the counter module (using shift register, pushbutton and 7 segment) is on.
  3. Traffic lights controller module is on and the Orientation indicator module based on an accelerometer is on.

When the system is powered, the traffic lights controller module works straight away and the 7 segment displays t. When the left side push button registers a button normal press, the second mode starts working. The push button module is sampled every 17ms in the code.


In the second mode the traffic lights controller will still be working. In this mode, the counter module and the right push button module will start working as well. When the right side pushbutton is pressed after 300ms the counter variable is increased and a different number or letter is displayed on the 7 segment display.


In the third mode, the counter module stops working, the traffic lights controller module is still working and the orientation indicator module is now switched on. The orientation indicator module displays letter "F" on the 7 Segment display indicating that the system is "FLAT", and displays letter "L" on the 7 segment indicating that the system is on "LANDSCAPE".

SYSTEM HARDWARE USED

  1. ATMEGA328P Microcontroller.
  2. 6 LED's and Resistors.
  3. 2 Push Buttons
  4. MPU_6050 Integrated Circuit.
  5. Shift register
  6. 7 Segment Display.
  7. Jumper cable wires.

The skills gained in this project can be applicable in developing systems in the following industries:

  1. Automotive systems. Modern automobiles are equipped with numerous embedded systems that need to operate concurrently in a non-blocking fashion.
  2. Industrial Automation. Some manufucturing facilities use embedded systems to control complex processes. Skills gained in this project, can help to develop code that can manage tasks like machine control and communication in a non-blocking fashion.
  3. Robotics. Robots with multiple actuators and sensors need to perform tasks concurrently such as path planning, motion control with the environment. Skills gained while doing this project, can be used to develope code that can run robot tasks in a non-blocking fashion.
